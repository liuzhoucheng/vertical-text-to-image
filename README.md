# h2c-vertical-typesetter

Deterministic `vertical-rl` CJK text rendering for [`html2canvas`](https://html2canvas.hertzen.com/) captures.

This library helps when a page looks correct in the browser, but the image generated by `html2canvas` loses vertical Chinese/Japanese/Korean text layout, rotates ASCII characters incorrectly, misplaces numbers such as `(2025)`, or fails to capture text from `textarea` / text inputs.

## Features

- Captures vertical Chinese/CJK text into a canvas image.
- Supports `writing-mode: vertical-rl`.
- Keeps CJK characters upright.
- Rotates ASCII letters, digits, punctuation, and compact numeric runs sideways.
- Handles year-like fragments such as `(2025)` more compactly.
- Reads text from normal elements, `textarea`, and text inputs.
- Provides a live preview canvas API.
- Provides pointer-to-text-index hit testing for custom editors.
- Supports optional style ranges: font size delta, bold, underline, and color.
- Keeps the original DOM mostly untouched during export.

## Install

```bash
npm install h2c-vertical-typesetter html2canvas
```

Or use the source file directly in a browser project.

## Quick Start

```html
<div id="paper">
  <div id="verticalText" style="writing-mode: vertical-rl; font-size: 26px; line-height: 52px;">
    福主：众多  参查来课：乙巳(2025)年农历八月初四日午时...
  </div>
</div>
```

```js
import html2canvas from "html2canvas";
import { captureWithVerticalTypeset } from "h2c-vertical-typesetter";

const canvas = await captureWithVerticalTypeset(document.querySelector("#paper"), {
  html2canvas,
  backgroundColor: "#fff",
  scale: 2,
});

document.body.appendChild(canvas);
```

## Browser Demo

Open `demo/index.html` directly in a browser, or run it through a local web server. For example:

```bash
npx serve .
```

Then visit the printed local URL and open `/demo/`.

The demo uses `dist/h2c-vertical-typesetter.global.js`, so the live vertical preview works even when opened through `file://`. The PNG download button still needs `html2canvas`; the demo loads it from jsDelivr by default.

The demo shows:

- A vertical `textarea` editor.
- A deterministic canvas preview.
- Selection/caret painting.
- Sample bold, size, and underline style ranges.
- PNG export through `html2canvas`.

## Preview API

```js
import { paintVisibleVerticalText } from "h2c-vertical-typesetter";

paintVisibleVerticalText(canvas, verticalBox, text, {
  selectionStart: 12,
  selectionEnd: 20,
  showCaret: false,
  styleRanges: [
    { start: 12, end: 20, fontDelta: 2 },
    { start: 12, end: 20, bold: true },
    { start: 12, end: 20, underline: "double" },
  ],
});
```

## Hit Testing

For custom text editors, browser-native vertical `textarea` selection may drift away from a custom canvas preview. Use:

```js
import { getVerticalTextIndexFromPoint } from "h2c-vertical-typesetter";

const index = getVerticalTextIndexFromPoint(verticalBox, text, event.clientX, event.clientY, {
  styleRanges,
});
textarea.setSelectionRange(index, index);
```

## Style Ranges

`styleRanges` are plain text-index ranges:

```js
[
  { start: 0, end: 4, fontDelta: 2 },
  { start: 4, end: 8, bold: true },
  { start: 8, end: 12, underline: "dotted" },
  { start: 12, end: 16, color: "#b91c1c" }
]
```

Supported underline values:

- `"solid"`
- `"double"`
- `"thick"`
- `"dotted"`
- `"dashed"`
- `"dashdot"`
- `"wavy"`
- `null`

## Capture Options

`captureWithVerticalTypeset(root, options)` accepts normal `html2canvas` options plus:

| Option | Type | Description |
| --- | --- | --- |
| `html2canvas` | function | Required unless `window.html2canvas` exists. |
| `styleRangesResolver` | function | Return style ranges for each handled vertical element. |
| `verticalSelector` | string | CSS selector used to collect vertical elements. Defaults to `"*"`. |
| `shouldHandleElement` | function | Return `false` to skip a vertical element. |
| `extraTopOffset` | number/string/function | Optional top offset for special templates. |
| `sidewayNumericRatio` | number | Font ratio for compact numeric runs. Defaults to `0.86`. |
| `previewIgnoreSelector` | string | Elements ignored during export, such as preview canvases/toolbars. |

## Chinese Note / 中文说明

这个库来自一个真实中文竖排日课图片生成场景。它解决的核心问题是：

- 页面里竖排文字显示正常，但 `html2canvas` 生成图片后错位。
- 中文、括号年份、数字、英文字母、标点混排后，导出图与编辑预览不一致。
- `textarea` 内容在竖排环境下选择和导出都不稳定。

开源版只保留通用的图片生成与竖排排版能力，不包含原业务页面、PHP、数据库、Logo、联系方式或私人内容。
